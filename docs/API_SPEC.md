## 1. Overview

The Bakery Timer API is a REST API built with FastAPI. It exposes two endpoints under the `/api` prefix. Full interactive documentation is auto-generated by FastAPI and available at `http://localhost:8000/docs` when the server is running.

**Base URL (local):** `http://localhost:8000`

**Content type:** All requests and responses use `application/json`.

---

## 2. Endpoints summary

| Method | Path | Description | Requirements |
|--------|------|-------------|--------------|
| `POST` | `/api/sessions` | Save a completed focus session | FR-16 |
| `GET` | `/api/sessions` | Retrieve all completed sessions | FR-17 |

## 3. Endpoint details

### 3.1 `POST /api/sessions`

Saves a completed focus session to the database. This endpoint is called by the frontend when the user closes the completion pop-up (FR-12).

**Cancelled sessions must never be sent to this endpoint** — that logic is enforced on the frontend (FR-07).

---

**Request**

```
POST /api/sessions
Content-Type: application/json
```

```json
{
  "cake_name": "Cupcake",
  "duration_minutes": 20,
  "completed_at": "2025-05-21T08:44:39Z"
}
```

**Request body fields**

| Field | Type | Required | Validation | Description |
|-------|------|----------|------------|-------------|
| `cake_name` | string | No | 1–100 characters | Name of cake type or custom name. Defaults to `"Custom"` if omitted |
| `duration_minutes` | integer | Yes | > 0 | Session duration in whole minutes |
| `completed_at` | string (ISO 8601) | Yes | Valid datetime | UTC timestamp of session completion |

---

**Responses**

**`201 Created` — session saved successfully**

```json
{
  "id": 3,
  "cake_name": "Cupcake",
  "duration_minutes": 20,
  "completed_at": "2025-05-21T08:44:39Z"
}
```

**`422 Unprocessable Entity` — validation failed (FR-18)**

Returned when required fields are missing, types are wrong, or values fail validation rules.

```json
{
  "detail": [
    {
      "type": "greater_than",
      "loc": ["body", "duration_minutes"],
      "msg": "Input should be greater than 0",
      "input": -5
    }
  ]
}
```

**`500 Internal Server Error` — unexpected server error**

```json
{
  "detail": "Internal server error"
}
```

---

### 3.2 `GET /api/sessions`

Returns all completed sessions, ordered by most recent first (FR-15).

---

**Request**

```
GET /api/sessions
```

No request body. No query parameters in v1.0.

---

**Responses**

**`200 OK` — sessions returned (including empty list)**

```json
[
  {
    "id": 3,
    "cake_name": "Cupcake",
    "duration_minutes": 20,
    "completed_at": "2025-05-21T09:30:00Z"
  },
  {
    "id": 2,
    "cake_name": "My deep work block",
    "duration_minutes": 40,
    "completed_at": "2025-05-21T08:44:39Z"
  },
  {
    "id": 1,
    "cake_name": "Cookie",
    "duration_minutes": 10,
    "completed_at": "2025-05-21T07:00:00Z"
  }
]
```

Returns `[]` (empty array) if no sessions exist — never `null`.

**`500 Internal Server Error`**

```json
{
  "detail": "Internal server error"
}
```

## 4. Response object schema

Both endpoints return session objects in the same shape:

| Field | Type | Description |
|-------|------|-------------|
| `id` | integer | Auto-assigned unique identifier |
| `cake_name` | string | Name of the cake / session |
| `duration_minutes` | integer | Duration in minutes |
| `completed_at` | string (ISO 8601) | UTC completion timestamp |

## 5. Error handling contract

The frontend must handle these cases gracefully (NFR-08):

| Scenario | Expected frontend behavior |
|----------|--------------------------|
| `POST` returns `422` | Show inline error; do not clear the timer |
| `POST` returns `500` | Show inline error "Could not save session. Please try again." |
| `GET` returns `500` | Show inline error "Could not load history." |
| Network timeout / no server | Show inline error "Cannot connect to server. Is the app running?" |

## 6. CORS configuration

The FastAPI app must include `CORSMiddleware` to allow browser requests from `localhost`. Without this, browsers block `fetch()` calls from the frontend to the API.

```python
from fastapi.middleware.cors import CORSMiddleware

app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:8000", "http://127.0.0.1:8000"],
    allow_methods=["GET", "POST"],
    allow_headers=["Content-Type"],
)
```

## 7. Route handler implementation (reference)

```python
# routes/sessions.py
from fastapi import APIRouter, Depends, status
from sqlalchemy.orm import Session as DBSession
from .. import models, schemas
from ..database import get_db

router = APIRouter(prefix="/api/sessions", tags=["sessions"])

@router.post("", response_model=schemas.SessionResponse, status_code=status.HTTP_201_CREATED)
def create_session(payload: schemas.SessionCreate, db: DBSession = Depends(get_db)):
    session = models.Session(**payload.model_dump())
    db.add(session)
    db.commit()
    db.refresh(session)
    return session

@router.get("", response_model=list[schemas.SessionResponse])
def get_sessions(db: DBSession = Depends(get_db)):
    return db.query(models.Session).order_by(models.Session.completed_at.desc()).all()
```
